%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% 複雑な例
% 優先度: 必須
% 以下のような場合は、
% <SaveBGM>
% <ReplayBGM>
% <Fadeout>
% の3つがタグとして認識される。
% 逆に、<StopSE>と<Fadein>の2つはタグとしては認識されない。
% つまり、
% BGMの保存コマンドとBGSの再開コマンドのが2つ挿入された後、
% ``メガ粒子〜
% レクイエム〜<StopSE>
% 青葉はじめひふみゆん
% シュート！(ﻭ ･ㅂ･)ﻭ ̑̑<Fadein>``
% という4行のメッセージが挿入され、フェードアウトコマンドが挿入される。
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
<SaveBGM>
<ReplayBGM>メガ粒子〜
レクイエム〜<StopSE>
青葉はじめひふみゆん
シュート！(ﻭ ･ㅂ･)ﻭ ̑̑<Fadein>
<Fadeout>


%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
% メッセージに影響を与えるタグとメッセージの間にその他のタグが入った場合
% 優先度: 低
% あまりいい例ではないが、以下のような場合は、
% 1つ目のメッセージがウィンドウ位置Middleとして挿入
% 2つ目のメッセージがウィンドウ位置Topとして挿入
%% 【注意】なお、優先順位は高くないので、<WindowPosition: Top>はキャンセル
%%  されてもよい。実装難易度がキャンセルされたほうが楽ならそちらでも良い
%% その場合はこのテストケースは削除する
%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%
<WindowPosition: Middle>今日も一日がんばるぞい！(و ・ㅂ・)و ̑̑

<WindowPosition: Top>
<comment>この辺でいい感じのBGMを挿入</comment>
今日も一日がんばるぞい！(و ・ㅂ・)و ̑̑
